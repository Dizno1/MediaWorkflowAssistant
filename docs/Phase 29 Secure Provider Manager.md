# Phase 29 Secure Provider Manager

## Status

Completed.

## Purpose

Phase 29 adds an accessible copy-and-paste provider configuration experience under Advanced assistance settings. Provider choice remains automatic during normal workflows. Technical settings are exposed only when the user deliberately opens the advanced section.

## Secure local storage

OpenAI, Azure OpenAI, and Google Gemini credentials are stored in IndexedDB. A non-exportable AES-GCM key generated by the Web Crypto API encrypts each provider configuration before storage. Credentials are tied to the current browser profile and computer. They are not written into project source, project ZIP files, Shared Knowledge, workflow history, publication packages, or Git.

Clearing browser site data removes the encryption key and saved credentials. Copying the repository to another computer does not copy credentials.

## Provider controls

Each provider includes labeled fields, a save action, a connection test, and a clear action. Password fields are emptied after saving. Existing saved keys are preserved when a user updates only an endpoint, deployment, or model field.

Supported configuration:

- OpenAI API key, transcription model, and vision model.
- Azure OpenAI resource endpoint, deployment name, and API key.
- Google Gemini API key and model name.
- The existing custom connected JSON service remains available for specialized endpoints.

## Automatic selection

The AI Provider Layer continues to select providers by capability, privacy, cost category, and quality. Users receive privacy and possible-cost confirmation before source content is sent externally. Provider settings remain an implementation detail unless an advanced override is selected.

## Accessibility

All fields use explicit labels. Save, test, and clear controls are keyboard operable. Connection results and errors are announced through the existing polite status region. No focus is moved unexpectedly. Provider availability is exposed in the existing provider-choice controls.

## Limitations

Browser encryption protects saved credentials at rest from casual inspection and prevents them from appearing in repository files. It does not protect against malicious code running in the same browser origin while the application is open. For stronger operating-system-backed protection, a future packaged desktop version should use Windows Credential Manager or the equivalent platform keychain.

Azure OpenAI capabilities depend on the deployments enabled for the configured resource. Gemini and OpenAI capabilities depend on account access, model availability, provider limits, browser cross-origin policy, and current service pricing.

## Testing

1. Start the application through a local web server.
2. Open Advanced assistance settings.
3. Paste a provider key and required endpoint or model values.
4. Save the provider and confirm that the password field clears.
5. Reload the page and confirm that the provider is reported as available without re-entering the key.
6. Run Test connection and confirm the result is announced.
7. Clear the provider and confirm that it becomes unavailable.
8. Inspect the repository and exported project packages to confirm that no credential values are present.
