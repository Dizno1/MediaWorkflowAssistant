# Phase 17 - Human Review and Approval

Phase 17 adds project-aware human review without replacing workflow execution, Shared Knowledge, or the Project Workspace.

## What changed

- Completed transcript, caption, audio-description, and accessibility-package workflows automatically create review records.
- Reviewers can be assigned in plain language.
- Review comments are retained with author and date information.
- Work can be approved, rejected with required revision guidance, or returned to pending.
- Every assignment, comment, and decision is retained in revision history.
- Project status distinguishes Review Required, Revision Required, and Ready to Publish.
- Project accessibility intelligence reports pending and rejected reviews and prevents a project from appearing publication-ready before required approvals are complete.

## Accessibility

The review workspace uses native forms, labels, buttons, headings, details elements, visible status text, and polite live-region announcements. All controls are keyboard operable and decisions are not communicated by color alone.

## Privacy

Review data is stored with the project in browser local storage. The application does not send assignments, comments, or decisions to an external service.
