Source Input Requirements

The Open Door Accessible Assistant must eventually accept content from two source types.

Local file

The user can choose or drop a file from their device.

Web address

The user can paste a URL for media or a page containing media, including services such as YouTube.

The application must identify the source and content type, determine what it can access, and present the same goal-based choices used for local files.

Users should not need to understand direct media links, download tools, codecs, platform APIs, or authentication methods.

URL input is a planned feature and is not implemented in Sprint 5 Part 1.
